# Master Prompting Template using the PRD Method

## 1. Core PRD Template Structure
- **P**roblem: Define the core problem the AI model solves.
- **R**equirements: List the essential requirements needed for the AI model to address the problem effectively. 
- **D**eliverables: Identify what the end output or result looks like.

## 2. Specific Instantiations for Levels 1, 2, and 3
### Level 1: 
- **Problem:** Identify a simple task that can be accomplished by the AI in a controlled environment.
- **Requirements:** Basic input data types; defined outputs; no complex decision-making.
- **Deliverables:** A straightforward response that aligns with the task requirements.

### Level 2: 
- **Problem:** Address a moderate complexity task requiring context understanding and basic inference.
- **Requirements:** More varied input data; some flexibility in outputs; basic feedback loop.
- **Deliverables:** A contextual response that uses relevant background information.

### Level 3: 
- **Problem:** Solve complex tasks that require strategic thought and multi-step reasoning.
- **Requirements:** Diverse input; sophisticated algorithms; continuous learning loops and adaptability.
- **Deliverables:** A comprehensive output considering multiple factors and potential outcomes.

## 3. Best Practices and Tips
- Start simple before moving to complex problems.
- Clearly define your problem and solutions to avoid scope creep.
- Use examples and case studies in your templates to provide clarity.
- Always include a feedback mechanism to refine prompts.

## 4. Framework for Adapting the Template for Higher Complexity Levels
- For levels 4 and 5, augment the PRD template by introducing:  
  - **Extended Problem Definition:** Involve multiple stakeholders to capture all aspects.
  - **Complex Requirements Gathering:** Engage in iterative requirement gathering with user feedback.
  - **Dynamic Deliverables:** Ensure deliverables can evolve based on continuous evaluation and user interaction.
